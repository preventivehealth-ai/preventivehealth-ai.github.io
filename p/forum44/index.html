Internal · Engagement Strategy

How we'll engage with Forum Health — without becoming their web agency.

A time-boxed Healthcare Marketing Infrastructure Stabilization engagement, followed by an optional lightweight gatekeeper retainer. Every deliverable is either configuration/governance work that lives in Forum Health's environment, or reusable IP we apply to future healthcare customers. No bespoke software is written for Forum Health.

Forum Health, a healthcare provider with fragmented marketing properties across multiple CMS platforms and hosting environments, has approached us for help consolidating their infrastructure, fixing their security and compliance posture, and exiting their incumbent vendor (Steer Health). We are a health technology company, not a web agency — and this strategy is built so we can play a pivotal role with Forum Health without drifting into bespoke software work for a single partner.

02 · Situation

What we're walking into

Five problem clusters. Two of them are framed in Forum Health's own materials as routine cleanup; both are materially more serious than that.

IF

Infrastructure fragmentation

  • Main site on WordPress 6.8.5 + Elementor Pro (Cloudways)
  • Hormones by Design on Steer's Next.js stack
  • Two sub-brand sites on legacy PHP/Apache/AWS
  • Some properties already on WP Engine
SEC

Security posture is inadequate

  • Main site recently hacked (defaced)
  • Root cause: absent identity / access lifecycle
  • Two sub-brand sites with no SSL, no WAF
  • No SSO, MFA, audit logging, or RBAC
PII

Active HIPAA exposure

  • Orphaned Meta Pixel firing on main site
  • Forum Health does not control the Meta Business Manager
  • Patient data may already be flowing to a third party
  • Same fact pattern as Novant / Advocate enforcement
AN

Broken analytics

  • Conversion values inflated 21x
  • 18+ external script domains per page
  • Two parallel GA4 properties + legacy UA
  • Microsoft Clarity + Hotjar redundantly running
GOV

No change governance

  • Operations staff making unvetted production edits
  • Tracking breaks repeatedly
  • Heather has mandated a hard stop on unapproved changes
  • Jade & Tanya are competent but lone single points of failure
VND

Vendor exit pressure

  • Forum Health has stated intent to exit Steer Health
  • Steer is embedded in marketing + clinical + ops
  • Hormones by Design is locked into Steer's stack
  • See separate Steer Exit Plan

What Forum Health's own materials understate

The Meta Pixel is a potential HIPAA disclosure event, not a "tag bloat" item. The 2022–2024 hospital settlements (Novant, Advocate Aurora, Sentara) are precedent. The hack was a process failure, not a credential failure — the root cause is an absent identity/access lifecycle. A WP Engine migration without solving this just relocates the vulnerability.

03 · Strategic Principles

Five rules that bound every decision

If a proposed activity violates one of these, the answer is no — regardless of revenue.

01

We do not write software for Forum Health

Configuration, integration, governance, audit, advisory — yes. Custom code — no.

02

Everything reusable becomes our IP

Security baseline, tag governance, deploy-gate runbook. First deployment for Forum Health, second deployment in days for the next customer.

03

Jade and Tanya own the steady state

Every workstream pairs us with one of them. By engagement end, they — not us — run the systems.

04

Capped hours, fixed fees, exit ramps

No T&M drift. If scope is wrong mid-engagement, we walk with a clean reputation and a paid phase delivered.

05

Steer's clinical scope is out of bounds

The Steer exit is sequenced separately. Clinical extraction is advised, never implemented.

04 · Alternatives Considered

Four engagement shapes — and why we picked one

Each evaluated against two goals: be pivotal to Forum Health, do not stretch drkai thin. Click any option to see the analysis.

Alternative A · Compliance & Infrastructure Authority

Recommended

We own four things for the marketing surface, end-to-end:

  1. Hosting consolidation.
  2. Marketing-surface security architecture — edge, identity, and application security for the WordPress properties and marketing tooling. Explicitly not clinical-system, endpoint, facility, or enterprise-IT security, which stay with Forum Health's IT function and EHR vendor.
  3. Marketing-surface patient privacy compliance — the web/tracking/lead-form layer. Explicitly not the EHR's clinical PHI scope, which stays with Forum Health's EHR vendor and clinical IT function.
  4. Marketing-surface change governance — written change classification, approval workflow, and technical enforcement (branch protection, GTM versioning, RBAC, audit logging) for the marketing properties and marketing tooling. Explicitly not enterprise IT change management, clinical workflow change management, or HR/financial-system change control.

For any eventual UX rebuild we vet, contract, and supervise an external agency partner we already work with — we do not execute the rebuild ourselves.

Pros

  • Plays directly to our healthcare-technology strengths
  • Sticky relationship — survives leadership changes
  • Defangs "can you build it too?" with a pre-defined answer
  • Productized deliverables compound across customers

Cons

  • Requires a vetted agency partner relationship
  • Requires discipline to refuse scope-creep into dev

Alternative B · Stabilization Sprint Only

Tactical

A fixed-scope 60–90 day engagement: migration, security lockdown, tag remediation, governance setup. Then we step back entirely.

Pros

  • Lowest commitment, fast credibility win
  • Easy to staff

Cons

  • No recurring revenue
  • Lose relationship gravity exactly when trust peaks
  • "Phase 2 TBD" usually means Phase 2 goes elsewhere

Useful as the first 90 days inside Alternative A, not as the whole engagement.

Alternative C · Full Technical Takeover

Not Recommended

We become Forum Health's web agency, executing both the infrastructure work and the multi-year UX rebuild.

Pros

  • Maximum visibility
  • Maximum revenue

Cons

  • Inverts our strengths
  • Requires UX/visual design hires against the drkai roadmap
  • Competing with specialists on their home turf
  • Multi-year rebuilds slip; blame attaches to the lead vendor

Alternative D · Audit & Advise

Optionality

We deliver a paid security + compliance audit and a remediation roadmap, then walk away.

Pros

  • Lowest risk
  • Lowest demand on our time

Cons

  • Lowest revenue
  • Hands trust we built to whoever executes the roadmap

Only viable if we're capacity-constrained and want to preserve optionality.

07 · Stabilization Sprint Workstreams

Four parallel workstreams

Each has a named lead on our side and a designated pair on Forum Health's side who owns it after we leave.

Lead: Infrastructure Engineer Pair: Jade Cabral

2A — Hosting consolidation onto WP Engine

  • BAA executed with WP Engine
  • Environment setup: dev / staging / production for the main site
  • Migrate forumhealth.com off Cloudways (clean — not deeply embedded in Steer's tooling)
  • Migrate proactivwellnesscenters.com and pwconline.com as separate installs (multisite is fragile under acquisition churn)
  • 301 redirect map for every migrated site — fixing the prior agency's redirect failure
  • Domain, DNS, and email-routing audit during cutover
Out of scope: Hormones by Design (stays on Steer's Next.js until the Steer exit plan addresses it). Any Shopify/ecommerce restoration.
Lead: Security & IAM Engineer Pair: Jade Cabral Productized IP

2B — Healthcare Marketing Security Baseline

Build the reusable baseline we deploy to every future healthcare customer.

  • SSO (Google Workspace or Okta) for all WP admin access; MFA enforced
  • RBAC defined for marketing vs. ops vs. external vendors
  • Audit logging via WordPress plugin with log forwarding to a central destination
  • WAF rule baseline — consolidating on Cloudflare with Page Shield (see WAF decision memo)
  • Secrets rotation schedule
  • Written incident response runbook — the protocol the anime-ad hack should have followed
Lead: Analytics / MarTech Engineer Pair: Jade Cabral Productized IP

2C — Tag, analytics, and consent governance

  • Full audit of the 18+ external scripts; kill list with per-tag rationale
  • Consolidate heat-mapping (keep Clarity, retire Hotjar)
  • Retire Universal Analytics and the duplicate GA4 property
  • Matomo coordinated with Steer explicitly — not ripped silently
  • Server-side GTM for conversion events — durable fix for the 21x inflation
  • Consent management platform (Cookiebot or OneTrust) configured for healthcare
  • Tag governance protocol with technical enforcement (PR-based versioned GTM workspace)
Lead: Engagement Lead Pair: Heather Spangler

2D — Change governance (Heather's actual request)

  • Branch protection on WP Engine staging → production
  • Written change approval protocol: routine content (Tanya can ship) vs. code/template/tag changes (review required)
  • Quarterly access review checklist
  • Weekly automated digest of changes deployed and access changes made

11 · Decision Required

Open questions before we commit

Two questions before the first call with Heather. The first shapes the immediate triage; the second sets the timing of the Steer Exit Plan relative to this engagement.

1
Is the Meta Business Manager controlling the orphaned Pixel reachable by anyone at Forum Health? If yes, we kill it at source; if no, the GTM kill needs a documented record for breach counsel.
2
How locked in is Forum Health to exiting Steer, and on what timeline? Determines whether the Steer Exit Plan runs in parallel with this engagement or after it.

One-paragraph version for Heather

We will stabilize Forum Health's marketing infrastructure in 90 days: kill the active compliance exposures in week one, consolidate hosting onto WP Engine with a healthcare-grade security baseline, fix the analytics, and put a written change-governance protocol in place that Jade and Tanya can run after we step back. We are not a web agency and will not run the multi-year UX rebuild — when that starts, we'll help you select an agency partner and we'll act as the production deploy gate so security and compliance never depend on the agency's discipline. The Steer exit runs as a separate, sequenced plan; we lead it on the marketing side and advise on the clinical side, with explicit conflict disclosure where our own product overlaps with replacement candidates.
